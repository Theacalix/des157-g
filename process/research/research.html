Research on Suitable Libraries for Fantasy Map Creation Project

Since my project is an interactive map generator, I looked for javascript libraries or examples that were related to what I want to make so that I could research the form instead of the content of my project. I initially started looking for javascript libraries related to interactive maps but discovered that most of them are geared towards real-world maps and data representations as opposed to fantasy based maps, which doesn’t work for my project.

For example, this article talks about a data mapping project that projects on top of a map of the city of Portland, and how they use javascript to change the map as users interact with it. However, I need something that can generate objects or images and allow users to place them on some sort of grid. So mapping libraries like Mapbox don’t seem that helpful to me. (It's very cool though and I used to live in Portland so check it out!)

I then realized that most of the maps used for dungeons are actually more similar to a floorplan or architects blueprints than a topographical outdoor map, so I next directed my research with that focus. I found this example, which is a floor plan made using the gojs library. The floorplan has a pannable window and each of the elements can be manipulated individually -very similar to what I’ll need to do. Gojs is a library for building “diagrams and graphs for your users, from simple flowcharts and org charts to highly-specific industrial diagrams, SCADA and BPMN diagrams, medical diagrams like genograms, and more.” They appear to have a nice structure for modeling different elements to create diagrams or blueprints. The floorplan example uses their model and node framework to store furniture, walls, and other architectural elements.

This is another example that I found, which uses the d3 library. This floor plan uses different layers to represent different things which can be toggled on and off. D3 is an alternative way to bind and access data, which follows all of the w3 standards and uses only HTML and CSS for visualization. It is an attempt to simplify some of the javascript needed to do things such as changing the color of all of your paragraphs for example. While not incredibly specific to my project, it could be helpful.

Lastly, while looking for information about floor plans, I found a discussion of the js canvas versus svgs. While I may end up using both, I definitely think making my assests like doors, walls, etc out of svgs will be helpful. Fabric.js is a library for increased svg support and creation allowing for users to interact with elements and manipulate them.

While I may end up using more than one of these, or even other libraries, currently gojs and fabric.js seem the most relevant to my project. Gojs seems to have a nice structure to help me organize the elements in my project and build needed dependencies. Whereas Fabric.js seems well geared towards the visual components of my project and some of the interaction that I will need.
